# Инструкции для разработчика

## Что нужно сделать

### 1. Изучить файлы
- Начните с README_GEMINI_ANALYZER.txt - краткий обзор
- Затем GEMINI_VIDEO_ANALYZER_PROMPT.txt - полное ТЗ
- GEMINI_ANALYZER_INTEGRATION_EXAMPLES.txt - примеры
- GEMINI_ANALYZER_SPECIFICATION_COMPLETE.txt - финальный отчет

### 2. Выбрать технологию
- Node.js + Express (рекомендуется)
- Python + FastAPI/Flask (альтернатива)

### 3. Создать API
- Endpoint: POST /analyze
- Принимает видео через multipart/form-data
- Анализирует с помощью Gemini 2.5 API
- Отправляет callback на наш сервер

### 4. Настроить интеграцию
- Callback URL: http://178.156.142.35:4123/api/analysis-callback
- Формат ответа: JSON с полями subtitles, voiceover, music_or_sfx, visual_effects, slides_or_text

### 5. Протестировать
- Локальное тестирование
- Интеграция с нашим сервером
- Проверка callback системы

## Ключевые моменты

### Gemini API
- Используйте Gemini 2.5 Pro
- Анализируйте только первые 5-10 секунд видео
- Обрабатывайте ошибки API

### Callback система
- Обязательно отправляйте результат на наш сервер
- Используйте retry логику
- Логируйте все операции

### Валидация
- Проверяйте типы файлов (mp4, avi, mov, mkv)
- Ограничивайте размер файлов (100MB)
- Валидируйте входные данные

## Контакты

При вопросах обращайтесь к создателю ТЗ.
Все необходимые данные для интеграции включены в файлы.
